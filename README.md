# Task Tracker

Тестовое веб-приложение для управления внутренними задачами компании.

Система позволяет создавать задачи, назначать исполнителей, изменять статус выполнения и отслеживать список заявок через веб-интерфейс.

---

# Используемый стек

## Backend

* Python 3.12
* FastAPI
* SQLAlchemy 2.0
* Pydantic v2
* SQLite
* Uvicorn

## Frontend

* HTML5
* CSS3
* Vanilla JavaScript (ES6 Modules)
* Fetch API
* Bootstrap 5

## Тестирование и инфраструктура

* Pytest
* HTTPX (TestClient для FastAPI)
* pytest-asyncio
* pytest-cov (покрытие кода)

# Реализованный функционал

## Работа с пользователями

* Получение списка пользователей
* Выбор исполнителя из списка при создании задачи
* Изменение исполнителя задачи

## Работа с задачами

* Создание задачи
* Получение списка задач
* Получение задачи по ID
* Фильтрация задач по статусу
* Изменение статуса задачи
* Изменение исполнителя задачи
* Удаление задачи

## Интерфейс

* Одностраничное модульное приложение (ES6 Modules)
* Таблица задач с цветовой индикацией статусов
* Фильтр по статусу
* Форма создания задачи
* Модальное окно просмотра и редактирования
* Автоматическое обновление данных после изменений
* Защита от XSS при рендеринге пользовательского ввода

# Запуск Frontend

⚠️ **Важно:** Из-за использования ES6-модулей (`type="module"`), файл `index.html` **нельзя** просто открыть двойным кликом в браузере (политика CORS заблокирует загрузку скриптов). Необходимо использовать локальный сервер.

Перейти в каталог frontend:

```bash
cd frontend
```

Запустить локальный сервер:

```bash
python -m http.server 5500
```

После запуска открыть в браузере:

```text
http://localhost:5500
```

Перед запуском фронтенда необходимо убедиться, что backend-сервер уже работает.

# 🧪 Тестирование

Проект покрыт интеграционными тестами API. Тесты используют **in-memory SQLite** (`sqlite:///:memory:`), что гарантирует изоляцию тестов от основной базы данных и высокую скорость выполнения.

## Структура тестов

```text
backend/tests/
├── conftest.py      # Фикстуры pytest и настройка тестовой БД
└── test_api.py      # Сценарии тестирования всех эндпоинтов
```

## Покрытие тестами

* ✅ Health-check эндпоинт (`GET /`)
* ✅ Получение списка пользователей (`GET /users`)
* ✅ Создание задачи с валидными данными (`POST /tasks`)
* ✅ Валидация данных при создании (ошибка 422)
* ✅ Получение списка задач с фильтрацией (`GET /tasks?status=...`)
* ✅ Получение задачи по ID (`GET /tasks/{id}`)
* ✅ Обработка несуществующей задачи (ошибка 404)
* ✅ Обновление задачи (`PUT /tasks/{id}`)
* ✅ Удаление задачи (`DELETE /tasks/{id}`)

## Запуск тестов

Перейти в каталог backend:

```bash
cd backend
```

Запуск всех тестов с подробным выводом:

```bash
pytest tests/ -v
```

Запуск с измерением покрытия кода:

```bash
pytest tests/ -v --cov=app --cov-report=term-missing
```

# Архитектурные решения

Backend построен по принципу разделения ответственности:

### Routers

Обработка HTTP-запросов и валидация входных данных.

### Services

Бизнес-логика приложения, независимая от транспорта.

### Schemas

Валидация и сериализация данных через Pydantic v2.

### Models

ORM-модели SQLAlchemy 2.0 с современным синтаксисом `Mapped`.

### Database

Подключение и управление базой данных через Dependency Injection.

### Tests

Интеграционные тесты с изолированной in-memory БД и переопределением зависимостей FastAPI.

Такое разделение упрощает поддержку проекта и позволяет масштабировать функциональность без усложнения кода.
