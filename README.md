# Task Tracker

Тестовое веб-приложение для управления внутренними задачами компании.

Система позволяет создавать задачи, назначать исполнителей, изменять статус выполнения и отслеживать список заявок через веб-интерфейс.

---

# Используемый стек

## Backend

* Python 3.12
* FastAPI
* SQLAlchemy 2.0
* Pydantic v2
* SQLite
* Uvicorn

## Frontend

* HTML5
* CSS3
* Vanilla JavaScript (ES6)
* Fetch API

---

# Реализованный функционал

## Работа с пользователями

* Получение списка пользователей
* Выбор исполнителя из списка при создании задачи
* Изменение исполнителя задачи

## Работа с задачами

* Создание задачи
* Получение списка задач
* Получение задачи по ID
* Фильтрация задач по статусу
* Изменение статуса задачи
* Изменение исполнителя задачи
* Удаление задачи

## Интерфейс

* Одностраничное приложение
* Таблица задач
* Фильтр по статусу
* Форма создания задачи
* Модальное окно просмотра и редактирования
* Автоматическое обновление данных после изменений

# Инициализация базы данных

Проект использует SQLite.

База данных создаётся автоматически при запуске приложения.

При первом запуске автоматически:

* создаются таблицы;
* добавляются тестовые пользователи.

Предзаполненные пользователи:

```text
Alex
alex@example.ru

John
john@yandex.ru
```

Дополнительная настройка базы данных не требуется.

---

# Запуск Backend

Перейти в каталог backend:

```bash
cd backend
```

Запустить сервер:

```bash
uvicorn app.main:app --reload
```

После запуска API будет доступно по адресу:

```text
http://127.0.0.1:8000
```

Swagger UI:

```text
http://127.0.0.1:8000/docs
```

---

# Запуск Frontend

Перейти в каталог frontend:

```bash
cd frontend
```

Запустить локальный сервер:

```bash
python -m http.server 5500
```

После запуска открыть в браузере:

```text
http://localhost:5500
```

Перед запуском фронтенда необходимо убедиться, что backend-сервер уже работает.

# Реализованные API эндпоинты

## Users

### GET /users

Получить список пользователей.

---

## Tasks

### POST /tasks

Создать новую задачу.

Пример запроса:

```json
{
  "title": "Bug fix",
  "description": "Fix login page",
  "requester": "Alex",
  "assignee_id": 1
}
```

---

### GET /tasks

Получить список задач.

Поддерживается фильтрация:

```text
/tasks?status=new
/tasks?status=in_progress
/tasks?status=done
/tasks?status=cancelled
```

---

### GET /tasks/{id}

Получить задачу по идентификатору.

---

### PUT /tasks/{id}

Обновить задачу.

Пример:

```json
{
  "status": "done",
  "assignee_id": 2
}
```

---

### DELETE /tasks/{id}

Удалить задачу.

# Архитектурные решения

Backend построен по принципу разделения ответственности:

### Routers

Обработка HTTP-запросов.

### Services

Бизнес-логика приложения.

### Schemas

Валидация и сериализация данных через Pydantic.

### Models

ORM-модели SQLAlchemy.

### Database

Подключение и управление базой данных.

Такое разделение упрощает поддержку проекта и позволяет масштабировать функциональность без усложнения кода.
